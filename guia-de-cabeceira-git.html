Arquitetura de Branches

Tipo Prefixo Origem Destino Finalidade
Feature feat/ develop develop Novas funcionalidades isoladas.
Fix fix/ develop develop Correção de bugs em desenvolvimento.
Release release- develop master/dev Homologação, QA e UAT.
Hotfix hotfix/ master master/dev Correção crítica em Produção.

Semântica e Commits

Padrão: tipo(escopo): descrição

  • feat: Nova funcionalidade (Gera MINOR)
  • fix: Correção de bug (Gera PATCH)
  • feat(api)!: Mudança Quebrante (Gera MAJOR)
git commit -m "fix(auth): corrige expiração do JWT (Ref: Card #442)"

A Release é o "Code Freeze".

  • QA: Validação técnica (Bugs).
  • UAT: Validação de negócio (PO).
  • Back-merge: Obrigatório após merge na master.

S.O.S - Panic Room

SITUAÇÃO DE CRISE: Mantenha a calma. Analise o impacto antes de agir.

Cenário: 1 PR entre 5 quebrou o sistema. Não dê rollback total!

git log --merges --oneline
git revert -m 1 <HASH_DO_PR>
git push origin master --tags

Cenário: Precisa de 1 feature da develop na release atual sem levar o resto.

git switch release-X.Y.Z
git cherry-pick <HASH_DO_COMMIT>

Cenário: A feature e seus fix/hotfix precisam sumir.

Regra LIFO: Reverte do mais novo para o mais antigo.

git revert -m 1 <HOTFIX>
git revert -m 1 <FIX>
git revert -m 1 <FEATURE>

Dicionário de Comandos

Comando Para que serve? Por que usar?
git fetch origin Atualiza referências Ver o que mudou sem alterar seu código.
git switch -c branch Cria/Troca branch Comando moderno, mais seguro que checkout.
git stash push -m "msg" Salva progresso temporário Trocar de contexto rápido para um hotfix.
git bisect Busca binária de bugs Achar exatamente qual commit quebrou o sistema.
git log --oneline --graph Visão da árvore Entender onde estão os forks e merges.
git show v1.0.0 Audita Tag Ver quem criou e o conteúdo da versão.
